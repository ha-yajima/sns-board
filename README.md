# 掲示板型SNS（チーム制作）

## 概要
ユーザー登録・ログイン・画像投稿・リポスト・返信機能を備えたSNSプラットフォーム。
職業訓練の課題として3名チームで制作。「初学者だからこそ、
Twitterのような普遍的な機能を持つものを作り、基礎を固める」
という方針のもと開発しました。

## 担当範囲（自分が主に実装した部分）
- ログイン・退会機能
- 返信機能（スレッド表示・画像添付対応）
- いいね・リポストのアイコン実装
- デザインコンセプトの立案・UI方針の提案

## 使用技術
- PHP 8.x（PDOを用いたセキュアなDB接続）
- MySQL（複雑なリレーションを持つDB設計）
- Ajax / Fetch API（非同期通信による「いいね」機能）
- HTML/CSS（レスポンシブ対応）

## 注力したポイント
- **返信機能のスレッド表示：** 投稿に対する返信を
  親子関係として管理し、@メンション付きで表示。
  DBのリレーション設計から実装まで担当しました。

  ![返信スレッド画面](画像)

- **データ取得のSQL設計：** 通常投稿とリポストを
  混合表示するためUNION句を使用。正確なデータ抽出を
  実現しました。
- **非同期通信の導入：** 「いいね」にAjaxを採用し、
  ページリロードなしでデータ更新できる設計にしました。
- **セキュリティ対策：** XSS対策（htmlspecialchars）・
  SQLインジェクション対策を実装しました。



## 苦労した点・解決プロセス

**削除済み投稿によるページング崩れ**
削除した投稿がDBのカウントに含まれたままになっており、
ページ送りの際に空白ページが発生する問題に直面しました。
デバッグ用のカウント確認コードを出力しながら原因を特定し、
AIも活用しながら、削除済みレコードを除外したカウント処理に
修正することで解決しました。

**Ajax「いいね」機能の実装**
ページリロードなしでいいね数を更新する非同期処理を
初めて実装しました。AIを活用しながら動作の仕組みを
理解し、自分のコードに落とし込みました。
